# 修复 Git 提交信息中文乱码

## 问题说明

之前的提交信息使用了错误的编码，导致在 GitHub 上显示乱码。

## 解决方案

### 方法一：使用交互式 Rebase（推荐）

1. 运行以下命令开始交互式 rebase：
   ```bash
   git rebase -i HEAD~4
   ```

2. 在编辑器中，将需要修改的提交前的 `pick` 改为 `reword`（或 `r`）

3. 保存并关闭编辑器

4. 对于每个标记为 `reword` 的提交，Git 会打开编辑器让您修改提交信息

5. 修改提交信息为正确的中文（使用 UTF-8 编码）

6. 保存并关闭编辑器

7. 完成所有修改后，强制推送到远程：
   ```bash
   git push -f origin master
   ```

### 方法二：逐个修改提交信息

#### 修改最近的提交（HEAD）

```bash
git commit --amend -m "配置 Git 编码和文件属性，修复中文显示问题"
git push -f origin master
```

#### 修改更早的提交

使用交互式 rebase：
```bash
git rebase -i HEAD~4
```

### 正确的提交信息（参考）

- 初始提交：`初始提交：路径搜索算法演示程序 - 包含Dijkstra、A*、RRT三种算法实现`
- 添加 README：`添加 README.md 项目说明文档`
- 配置编码：`配置 Git 编码和文件属性，修复中文显示问题`
- 添加配置脚本：`添加 Git 中文编码配置脚本和说明文档`

## 注意事项

⚠️ **重要警告**：
- 修改已推送的提交历史需要 `git push -f`（强制推送）
- 如果其他人也在使用这个仓库，强制推送可能会影响他们的工作
- 建议在修改前先备份或创建新分支

## 如果不想修改历史

如果不想修改历史提交，可以：
1. 保持现状（GitHub 上显示乱码，但不影响功能）
2. 在 README 中添加说明，解释提交历史
3. 未来的提交使用正确的编码即可
